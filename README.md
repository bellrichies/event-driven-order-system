# event-driven-order-system
A simulation of a logistics order and dispatch system built using event-driven architecture, similar to real-world delivery platforms.
This project models how orders flow through multiple stages—from creation to dispatch and fulfillment—using asynchronous communication.

Key Features
Order creation and lifecycle management
Event-based communication between services
Dispatch engine simulation
Rider assignment logic
Queue-based processing (Redis / RabbitMQ)
